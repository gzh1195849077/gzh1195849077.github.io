头文件的引入

VS2022的安装目录G:\new_soft\vs2022\ide\VC\Tools\MSVC\14.32.31326\include

如果是标准库头文件，那么既可以采用<>的方式，又可以采用” “的方式，而用户自定义的头文件只能采用” “的方式。

例如，如下这种方式是正确的：

  • 使用 “xxx.h”,告诉编译器,从当前工作目录开始查找
  • 使用<xxx.h>,告诉编译器,从系统默认目录中去查找
  • 当不确定的时候,就使用双引号,系统会从当前工作目录找完成后,然后再去系统默认目录中查找。若 #include “xxx.h” 查找成功，则遮蔽 #include <xxx.h> 所能找到的同名文件；否则再按照 #include <xxx.h> 的方式查找文件。另外标准库头文件都放在 #include <> 所查找的位置
